Euler Topological Planar Partition Explorer (ETP2E)

Parse Euler Topological Planar and enumerate stroke combinations by mode (Stroke-only, Slice-only, Stroke-then-Slice). Results are clipped to the gray canvas bounds and paginated at 20 cards/page.

How to Use

This website is mainly served for mathematician, topology mathematician, or those who are interested in topology. Assuming you have not prepare your SVGs, and assuming you did and what this mechanism does.

Uploading SVG

1. Use any SVG editor app. I personally use Adobe Illustrator. You can use Figma.
2. Create a stroke pattern. In Figma, create a frame first, define a square as equal to the frame. Then use either R (Rectangle), O (Ellipse), and L (Line) to define your strokes.
3. Export your stroke pattern to SVG. In Figma, select your frame, right click, and find "Copy/Paste As" and select "Copy as SVG".
4. Paste your given SVG to the TextArea right below.

Using ETP2E

- possible_strokes: defines how many strokes must be initialized. Stroke in this definition, refers something like this, imagine I told you to draw a line, you can draw a circle, and that's 1 stroke, but the moment you draw straight to the corner, that's 2 strokes already. Test it out.
- mode: S! (Stroke-Only, a stroke is defined as a each SVG path element. Example, if you have 6 lines on a simple square, then using S! you'll have 6 strokes),
C! (Cut only, a stroke is defined as a cut of all available strokes. Example: (0,0), (0,1), (1,0), and (1,1) dots. How many lines you can do? 6. But I didn't ask to write 2 strokes given 6. But more like, slice all the stroke, which leave you with a even minier stroke (I assume it should be 8 total stroke, since the one that crosses gives 4 mini stroke when cut).),
StC (Stroke-and-Cut, stroke is initially defined as each SVG path element, but for each generated SVG path element, we do C!)
Test it out.
- is_connected_stroke_only: In hypothetical geometry, it is possible to write a single stroke of line, where in 2d it will appear as they are separated.
Analogy: Imagine drawing a line on one face of a cube. At that face, you can take alternating direction by switching to another face of a cube, and then return to the same face. From 2d perspective, you just shifted, in 3rd perspective, you simply walked in the other way.
Example: Generate this current code, go to Page 30, and see for Generation #707. It is clearly separated, but this program flags it as a single stroke. By enabling this variable, you only allow those whose stroke are connected in 2 dimension.
- go_to_page: Allows you to go to any page.
- Generate (buton): To generate a new ETP.

- base_strokes: Defines the available stroke in your SVG element.
- theoretical_combinations: Defines the combinatoric value of a given each SVG element path. (Hard to explain, but so on. I will document an elementary proof to elaborate on how this formula can be achieved mathematically)
- generated_combinations: Similar to theoretical but limited to what can be generated. This can be highly influenced to variable such as: is_connected_stroke_only. NOTE: this is capped to 50.000 gens.
- scanned__candidates: it is generated_combination but without capping ability, showing you the actual amount
- time taken: Time taken to render.
